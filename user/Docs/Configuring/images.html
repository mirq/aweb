Using your own default images

Purpose

There are moments where just displaying text is not sufficient, but some imagery is needed. For some of these imagery, AWeb uses external images that you can replace with your own.

AWeb uses these external images in the following places:

  • As default icons for unloaded images
  • For displaying WWW icon entities, like "&document;" (&document;)

All external images are located in the Images drawer. These are image files that can be in any format for which a datatype is available. Transparent GIFs are also supported.

Unloaded images

For unloaded images, AWeb uses the following files:

file nameused forcurrent image
def_imageUnloaded normal image
def_imagemapUnloaded image map
def_errimageImage that couldn't be loaded.

Icon entities

For all supported icon entities there is a file with the same name. So for the "&document;" entity there is a file named "document" in the "Images" drawer.

Look at the overview for all supported entities.

Installing your own images

To install your own image, just copy your image over the old one. Be sure to use the same name, and don't use a suffix like .gif or .png.

The Storage drawer contains some alternatives for the icon entities, but you can use any image of course.
