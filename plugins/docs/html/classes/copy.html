AOTP_COPY class

Purpose

The AOTP_COPY class represents a displayed copy of a loaded file. It delegates the actual rendering and directly related tasks to the attached AOTP_COPYDRIVER object.

Supported methods

Method Description
AOM_SET Set attribute values

Supported attributes

Attribute Type Description Applicability
AOBJ_Changedchild struct Aobject * Let the AOTP_COPY object know that this object has changed its dimensions. Probably a relayout is needed. AOM_SET
AOCPY_Onimganim BOOL Let the AOTP_COPY object know that last frame of an animated GIF has been fully decoded and rendered.

Only to be used by image decoders.

AOM_SET
AOCPY_Onimgload BOOL Let the AOTP_COPY object know that the image has been fully decoded and rendered.

Only to be used by image decoders.

AOM_SET
AOCPY_Portnumber long Get the ARexx port number for the window in which this object is being displayed.

Supported by AWeb version 3.5 and higher.

AOM_GET

Usage

Whenever a AOTP_COPYDRIVER subclass object changes its dimensions, it must set the AOBJ_Changedchild attribute of the AOTP_COPY object that it belongs to. The value of this attribute is a pointer to the AOTP_COPYDRIVER subclass object.

The AOCPY_Onimgload and AOCPY_Onimganim attributes should only be set by image decoders. See the JavaScript onLoad event section in the plugin copydriver description.
